# TASK-001

## Summary
이메일/비밀번호와 투자 성향 정보를 입력받아 신규 회원을 등록하는 회원가입 기능을 제공한다.

## Background
StockPilot은 사용자의 투자 성향(위험 성향·투자 기간)에 기반한 개인화 추천을 핵심 가치로 삼는다. 이를 위해서는 서비스 이용의 시작점인 회원가입 단계에서부터 인증 정보와 투자 성향 정보를 함께 수집해야 한다. 로드맵 Phase 1(인증/회원)의 첫 단계로, 이후 로그인·추천 기능의 전제 조건이 된다.

## Goals
- 사용자가 이메일/비밀번호로 회원가입할 수 있다.
- 회원가입 시 위험 성향과 투자 기간 정보를 함께 등록할 수 있다.
- 비밀번호는 안전하게 저장되어 원문 노출 위험이 없다.
- 동일 이메일의 중복 가입을 방지한다.

## Scope

### In Scope
- 회원가입 기능(이메일, 비밀번호, 닉네임, 위험 성향, 투자 기간 입력)
- 입력값 검증
- 이메일 중복 검증
- 비밀번호 안전 저장
- 회원가입 결과 응답(식별자, 이메일, 닉네임)

### Out of Scope
- 로그인 및 JWT 발급
- 이메일 인증(본인 확인)
- 비밀번호 재설정/변경
- 소셜 로그인

## Requirements

### Functional
- 사용자는 이메일, 비밀번호, 닉네임, 위험 성향, 투자 기간을 입력하여 가입을 요청할 수 있다.
- 위험 성향은 공격형/안정형/배당형 중 하나여야 한다.
- 투자 기간은 단기/장기 중 하나여야 한다.
- 이미 등록된 이메일로는 가입할 수 없다.
- 필수 입력값이 누락되거나 형식이 올바르지 않으면 가입이 거부되어야 한다.
- 가입 성공 시 사용자 식별자, 이메일, 닉네임 정보를 확인할 수 있어야 한다.

### Non-Functional
- 비밀번호는 복호화가 불가능한 방식으로 저장되어야 한다.
- 이메일 중복 검사 및 저장 과정은 동시 요청 상황에서도 데이터 정합성이 보장되어야 한다.

## Acceptance Criteria
- [ ] 유효한 입력으로 가입 요청 시 사용자가 생성되고 식별자/이메일/닉네임을 확인할 수 있다.
- [ ] 이메일 형식이 올바르지 않거나 필수값이 누락된 경우 가입이 거부된다.
- [ ] 위험 성향 또는 투자 기간이 정의된 값 이외로 입력된 경우 가입이 거부된다.
- [ ] 이미 존재하는 이메일로 가입 요청 시 가입이 거부되고 사용자에게 중복 사실이 식별 가능하게 응답된다.
- [ ] 동일 이메일로 동시에 두 건의 가입 요청이 발생해도 최종적으로 하나의 계정만 생성된다.
- [ ] 저장된 비밀번호 값을 조회해도 원문 비밀번호를 알아낼 수 없다.

## Dependencies
None

## Test Requirements
- 단위 테스트
  - 유효한 입력에 대한 가입 성공 로직 검증
  - 이메일 형식/필수값 누락 등 입력 검증 실패 케이스 검증
  - 위험 성향/투자 기간 허용 값 이외 입력 시 실패 케이스 검증
  - 중복 이메일 가입 시도 시 실패 케이스 검증
  - 비밀번호가 평문과 다른 형태로 저장되는지 검증
- 통합 테스트
  - 회원가입 요청 end-to-end 시나리오(성공/검증 실패/중복 이메일) 검증
  - 동일 이메일 동시 가입 요청 시 데이터 정합성 검증

## Status
IN_REVIEW
