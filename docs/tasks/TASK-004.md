# TASK-002

## Summary
로그인한 사용자가 종목을 관심종목으로 등록/해제하고, 종목의 관심 등록 수(watch_count)가 동시 등록 상황에서도 정확히 집계되도록 한다.

## Background
로드맵 Phase 2(종목 · 관심종목) 및 기획서 4.2/6.3에 따라, 사용자가 관심종목을 관리할 수 있어야 한다. 다수의 사용자가 동시에 동일 종목을 관심종목으로 등록할 경우 관심 등록 수 집계에 갱신 손실(lost update)이 발생할 수 있으므로, 이를 방지하기 위한 동시성 제어가 반드시 포함되어야 한다.

## Goals
- 로그인한 사용자가 종목을 관심종목으로 등록/해제할 수 있다.
- 같은 사용자가 같은 종목을 중복 등록할 수 없다.
- 종목의 관심 등록 수가 동시 등록 상황에서도 정확히 집계된다.
- 사용자가 자신의 관심종목 목록을 조회할 수 있다.

## Scope

### In Scope
- 관심종목 등록, 해제, 내 목록 조회 기능
- 관심 등록 수 동시성 제어
- 관심종목 기능 동작에 필요한 최소한의 종목 정보 관리(등록/수정 등 마스터 관리 제외)

### Out of Scope
- 실시간 시세 연동
- 추천 로직
- 종목 마스터 데이터의 생성/수정/삭제 관리 기능

## Requirements

### Functional
- 인증된 사용자는 종목을 관심종목으로 등록할 수 있다.
- 인증된 사용자는 자신이 등록한 관심종목을 해제할 수 있다.
- 인증된 사용자는 자신의 관심종목 목록을 조회할 수 있다.
- 이미 등록된 관심종목을 동일 사용자가 다시 등록하는 것은 허용되지 않는다.
- 인증되지 않은 사용자의 요청은 거부되어야 한다.

### Non-Functional
- 동일 종목에 대한 동시 다발적 등록 요청 상황에서도 관심 등록 수 집계에 데이터 정합성이 보장되어야 한다.
- 동시성 충돌 발생 시 사용자 요청이 임의로 실패 처리되지 않고 최종적으로 정합성 있는 결과를 반환해야 한다.

## Acceptance Criteria
- [ ] 관심종목 등록 시 해당 종목의 관심 등록 수가 정확히 1 증가한다.
- [ ] 동일 종목에 대해 서로 다른 N명의 사용자가 동시에 등록을 요청해도, 관심 등록 수가 정확히 N만큼 증가한다.
- [ ] 이미 등록한 종목을 동일 사용자가 다시 등록 요청하면 거부되며, 이때 HTTP 상태 코드는 409이다.
- [ ] 관심종목 해제 시 해당 종목의 관심 등록 수가 정확히 1 감소한다.
- [ ] 인증되지 않은 사용자가 등록/해제/조회를 요청하면 HTTP 상태 코드 401로 거부된다.
- [ ] 사용자가 자신의 관심종목 목록을 조회하면 본인이 등록한 종목만 반환된다.

## Dependencies
- 인증/인가 체계(로그인한 사용자 식별) — 선행 구현 완료 상태로 가정

## Test Requirements
- 단위 테스트: 관심종목 등록 로직, 해제 로직, 중복 등록 거부 로직 검증
- 통합 테스트: 관심종목 등록/해제 End-to-End 시나리오, 인증되지 않은 요청에 대한 거부 시나리오 검증
- 동시성 통합 테스트: 다수의 스레드가 동일 종목에 대해 동시에 관심종목 등록을 요청했을 때, 관심 등록 수가 요청 수만큼 정확히 증가하는지 실제 데이터 저장소 기반으로 검증
